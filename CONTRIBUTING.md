# Contributing to Payment Pipeline

Thank you for your interest in contributing! This project demonstrates enterprise payment infrastructure concepts.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Payment-Pipeline.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature`
5. Make your changes
6. Run the dev server: `npm run dev`
7. Test your changes
8. Commit: `git commit -m "feat: your feature description"`
9. Push: `git push origin feature/your-feature`
10. Open a Pull Request

## Code Style

- Use TypeScript for all new files
- Follow existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic
- Keep components small and focused

## Commit Message Format

We use conventional commits:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `chore:` - Maintenance tasks
- `refactor:` - Code refactoring
- `style:` - Formatting changes
- `test:` - Adding tests

## Project Structure
